49 /* A signed pointer value embeds the original pointer together with
50  a signature that attests to the validity of that pointer. Because
51  this signature must use only "spare" bits of the pointer, a
52  signature's validity is probabilistic in practice: it is unlikely
53  but still plausible that an invalidly-derived signature will
54  somehow equal the correct signature and therefore successfully
55  authenticate. Nonetheless, this scheme provides a strong degree
56  of protection against certain kinds of attacks. */

190 /* Compute a signature for the given pair of pointer-sized values.
191  The order of the arguments is significant.
192 
193  Like a pointer signature, the resulting signature depends on
194  private key data and therefore should not be reliably reproducible
195  by attackers. That means that this can be used to validate the
196  integrity of arbitrary data by storing a signature for that data
197  alongside it, then checking that the signature is still valid later.
198  Data which exceeds two pointers in size can be signed by either
199  computing a tree of generic signatures or just signing an ordinary
200  cryptographic hash of the data.

203  not have as many bits of entropy as that type's width would suggest;
204  some implementations are known to compute a compressed signature as
205  if the arguments were a pointer and a discriminator.
206 
207  The arguments must be either pointers or integers; if integers, they
208  will be coerce to uintptr_t. */
